Quick Read

Takeaways

  1. Simple hello world program
#include <stdio.h>
#include <stdlib.h>
int main(void){
	puts("Hello, world!");
	return EXIT_SUCCESS;
}
  1. Compile and run the program with GCC
cc -o hello hello.c #compile hello.c into hello executable

./hello #run the hello executable*emphasized text*
  1. Preprocessor directive , i.e: #include <file.h>
    • In a C program, it is replaced by the header file contents(at the same place) when the C program is run.
  2. puts is declared in stdio.h
    • It writes to stdout a string in a new line
    • puts return a postive number i.e ‘0’ when its run successfully
    • puts returns a negative number i.e ‘EOF’ when there is an error
  3. EXIT_SUCCESS is declared in stdlib.h
    • Its an object like maro that expands to ‘0’ returned in host enviroments
  4. main function defines the entry point of the program program in a hosted execution enviroment
  5. C execution enviroments:
    • Hosted (Most commen OSs)
    • freestanding (embedded, probably bare-metal)
  6. void as an input parameter in the main function means it doesn’t take any inputs
  7. Spaces and tabs in a code in a C code block are generally for stylistic reasons only
  8. Any code after a return statments doesnt run
  9. Formatted Ouput with printf + returns the number of chars when successful, returns negative when errors + fprint has certain ‘conversion specfications’ like s for string literal i.e %s
printf("%s\n","Hello, world!");

  3. (Reference Point) Portability Issus between implementing different C standards, i.e ‘c99, c90,c17,etc’ in different operating systems as well:
    1. Implementation-defined behavior
      • different implementation of a standared concepts:
        • example: how many bits in a byte
    2. Unspecified behavior
      • Multiple ways of doing something
    3. Undefined behavior
      • Not specified in the standard
    4. Locale-specific behavior and common extensions
      • Cultural differances and different extension adoption
