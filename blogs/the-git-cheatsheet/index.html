Setting up a git repo locally

The easy way

  1. install VScode and setup git addon/plug-in

Short way !

  1. Make a folder through GUI
  2. copy link of git repo
  3. inside the folder, open git bash with right click, then type
git clone git@git....git

Now your git repo is set !

The long way

  1. Make a folder through GUI or terminal
# Making folder through terminal (optional)
#For Mac/Windows Git bash
mkdir newFolder 

#change directory to new folder (optional)

cd newFolder
  1. Initiating local git floder in “newFolder”
#NOTE: default branch now is 'master'
git init
  1. adding remote repo, and naming it locally as ‘origin’
#the link is supposed to be git repo link

git remote add origin git@git....git

Now your git repo is set !

How to add your changes to the repo (your routine!)

To edit/create a new file to be commited & pushed

  1. make sure you are in the correct folder (the git folder)
  2. make sure your local repo is up to date, pull any new changes
	#this terminal command does it
	# VScode may have other options
	git pull origin [yourbranchName]
  1. create a new branch
	#if you are using a bash terminal, it should be like this
	#name it after the feature you are adding 
	git checkout -b yourNewBranchName
  1. in the same git folder on your PC, add the file you want to add or change the file you want changed
  2. stage files
	#Either name files or run the following
	git add .
  1. commit the changes
	# as you can see, you can add a comment to the the commit (good practice)
	git commit -m "These are my changes to the repo !!"
  1. push the changes !
	git push -u origin yourNewBranchName
  1. BONUS!: do step 5-7 with one line
	# :)
	git add . ; git commit -m "chachaChanges" ; git push

Git helper bash scripts

you can add this to your ~/.bashrc or ~/.bash_profile on Mac or Unix-like operating systems

Track All New Remote Repo Branches


gitTrackAllRemote(){
	#tracks all new remote branches
	#git branch -r outputs list of remote branches
	# ${remote#origin/} is a parameter expansion in bash that removes 'origin/' from the results on each iteration
	# example of the command on each iteration : git branch --track newbranch origin/newbranch
	# the previous comment shows a command that makes a local branch and makes it track the remote branch origin/newbranch

	for remote in `git branch -r`; do git branch --track ${remote#origin/} $remote; done


}

Sync local git directory with remote repo: deleted remote branch(s)

Note: make sure your branches don’t have any quotes

gitSyncRemoteRemovesForce(){
	#removes local branches the have been removed from remote repo
	


	#git fetch -p is similar to:
	#git fetch --prune
	# git remote prune origin
	# What do these commands  do ?
	# They all remove the local references of remote branches
	# once the local reference of a removed remote branch is gone, git branch --vv can show that
	# it outputs a next to the local branch name "gone"
	# we can then then pass that list to a program called xargs 
	# that would iterate over the name and runs the command stated after it 
	git fetch -p && git branch -vv | awk '/: gone]/{print $1}' | xargs git branch -D



}
